# DevForge AI 对话高强度测试与回归验收清单

更新时间：2026-04-30

## 1. 目标

这份清单用于判断 DevForge AI 对话是否已经具备真实高强度使用条件。

重点不是只看“能不能回复”，而是验证：

- 大会话不会卡死。
- 长工具结果不会撑爆 DOM。
- 工具循环不会空转。
- Provider 差异不会破坏主链路。
- 历史恢复、压缩、继续生成、回退、分支会话能稳定组合使用。
- 出错时用户看到的是可理解、可恢复的提示，而不是协议细节或红框噪音。

## 2. 当前架构基线

当前已具备以下基础：

- AI 请求主链路走 `AiGateway`。
- chat / compact / prompt optimize 已具备统一 Gateway 接入基础。
- Provider Dialect 已开始区分 DeepSeek / MiMo / Kimi Code 等兼容差异。
- 工具循环已加入重复工具、探索空转、工具结果缺失等保护。
- 连续只读 / 搜索无正文时，会自动切到“基于已有工具结果总结”的收敛模式。
- 大工具结果默认摘要展示，完整结果按需打开。
- 长 Markdown 和流式长文本有轻量预览保护。
- 大会话历史恢复已有工具摘要与窗口化加载基础。

## 3. 测试环境

### 3.1 基础环境

- 操作系统：Windows 11。
- Node / pnpm：使用项目当前锁定版本。
- Rust：满足 `src-tauri/Cargo.toml` 的 `rust-version`。
- 启动方式：`pnpm tauri dev` 或当前团队标准启动命令。
- 测试分支：`main` 最新提交。

### 3.2 Provider 准备

至少准备以下 Provider：

- DeepSeek：`deepseek-v4-flash`、`deepseek-v4-pro`。
- MiMo：至少一个 `mimo-v2` 系模型。
- Kimi Code：至少一个代码模型。
- 一个故障 Provider：错误 key 或不可达 endpoint，用于 fallback 测试。

### 3.3 测试项目准备

准备三个工作区：

- 小项目：少于 100 个文件。
- 中项目：1000 到 5000 个文件。
- 大项目：超过 10000 个文件，包含较大日志、JSON、Markdown、前端组件和后端代码。

## 4. 阻断级别定义

### P0 阻断

出现任意一项必须停止发布：

- 页面卡死超过 10 秒且无法恢复。
- 输入框无法输入、回车无法发送、按钮无法点击。
- 大会话一打开即白屏或全局不可交互。
- 工具循环无限执行，无法自动停止或取消。
- API Key 明文异常泄露到消息正文、日志面板或截图可见区域。
- 误执行危险写入 / 删除 / 命令操作且没有确认。
- Provider fallback 使用了错误 Provider 的 API Key。

### P1 高风险

可以继续测试，但发布前必须修：

- 长回复滚动明显掉帧。
- 工具结果摘要不可理解，用户不知道下一步怎么做。
- 历史会话恢复超过 5 秒。
- 压缩后上下文明显丢失关键工具结果。
- Provider 切换后模型能力显示与实际调用不一致。
- 自动收敛后没有产出有效总结。

### P2 可迭代

不阻塞内测：

- 局部动画不顺。
- 文案不够精细。
- 某些诊断字段缺少 hover 解释。
- 个别测试场景性能还有优化空间但不影响可用。

## 6. 大会话与滚动压力测试

| 编号 | 场景 | 操作 | 预期 |
|---|---|---|---|
| B1 | 100 条消息 | 打开历史会话 | 2 秒内可交互 |

## 15. 发布门禁

### 可以进入内部高强度使用

满足：

- 自动化验证命令全部通过。
- P0 为 0。
- Provider Chat / Tool Calls 至少 2 家通过。
- 大会话 500 条消息可打开、可滚动、可继续输入。

### 可以进入小范围真实使用

满足：

- 连续 2 天高强度测试无 P0。
- P1 不超过 3 个，且都有明确规避方案。
- DeepSeek / MiMo / Kimi Code 至少各通过基础 Chat。
- 至少一个 Provider 完整通过 Tool Calls / JSON / 模型同步。

### 可以对外稳定发布

满足：

- 连续 1 周无 P0。
- P1 清零或全部降级为 P2。
- Provider 实测矩阵中 DeepSeek 全项通过。
- 大会话 1000 条消息回归通过。
- 压缩、收敛、fallback、继续生成、回退、分支会话组合测试通过。

## 16. 当前建议

当前 DevForge AI 对话可以开始内部高强度测试。

不建议直接宣称最终成熟，原因是：

- Provider 真实 API 矩阵还没完整跑完。
- 大会话、压缩、工具收敛、fallback 的组合场景还缺连续多日实测。
- `AiChatView` 和主对话链路仍是高风险区域，后续需要继续拆组件和加端到端测试。

下一步优先级：

1. 跑 DeepSeek 完整能力矩阵。
2. 用 500 到 1000 条消息的真实大会话压测。
3. 连续触发工具空转收敛，确认不再出现红色错误框。
4. 测试压缩后继续执行多文件修改。
5. 记录所有 P0 / P1，按阻断级别修复。
